**Voice & AI Infrastructure**
> Real-time bidirectional speech pipelines. LLM inference orchestration. WebSocket lifecycle management. The hard part isn't the AI — it's making the connection reliable at latency budgets that voice demands.

**High-Throughput Platforms**
> Multi-tenant systems built for serious concurrent load. RBAC with ancestry caching. Async task pipelines. Permission factory patterns. The kind of work where a bad index design surfaces at 2am, not in dev.

**Enterprise Integrations**
> Active Directory / LDAP auth at scale. Multi-company HRIS backends. Payroll, approval chains, modular architecture that survives an org reorg.

I don't publish my employer's architecture. I share what I *learned* building it.

## `~/deep-dives`

<details>
<summary><b>🗄️ PostgreSQL — what I've actually studied</b></summary>
<br/>

- B-tree structure, page layout, and split behavior
- MVCC, transaction isolation, and vacuum mechanics  
- `EXPLAIN ANALYZE` plan reading — hash vs merge vs nested-loop join selection
- Planner cost model, `pg_stats`, and when it lies to you
- Partial indexes, covering indexes, index-only scans
- Connection pooling with pgBouncer — session vs transaction mode tradeoffs
- The N+1 problem at ORM level: when `select_related` vs `prefetch_related` matters and why it shows up in SQL

</details>

<details>
<summary><b>⚡ Redis — beyond caching</b></summary>
<br/>

- Expiry strategy design and TTL as first-class architecture
- Key namespace patterns at scale
- Lua scripting for atomic multi-key operations
- Pub/Sub reliability limits — what it can't guarantee and why
- Role-ancestry caching patterns for RBAC systems
- Session revocation without a blacklist: cutoff-based JWT invalidation
- When sorted sets beat hash maps and when they don't

</details>

<details>
<summary><b>🔁 Celery — task systems that don't lose data</b></summary>
<br/>

- Task idempotency design — why it matters and how to enforce it
- Retry mechanics, exponential backoff, and max-retries traps
- Visibility timeout and message re-delivery — the silent failure mode
- `chord` and `chain` failure propagation
- Audit log pipelines via async tasks — fire-and-not-quite-forget patterns
- Beat scheduling, task deduplication, and lock-based serialization

</details>

<details>
<summary><b>🌐 Networking internals</b></summary>
<br/>

- TCP flow control, congestion avoidance (Reno, CUBIC)
- Connection pooling mechanics — why your "fast" service is actually TCP-bound
- HTTP/1.1 keep-alive vs HTTP/2 multiplexing
- WebSocket connection lifecycle, heartbeats, and teardown
- `SO_REUSEADDR`, `TCP_NODELAY`, `SO_KEEPALIVE` — when they matter
- `epoll` vs `select` vs `kqueue` — the I/O multiplexing model

</details>

<details>
<summary><b>🧠 Memory & concurrency</b></summary>
<br/>

- L1/L2/L3 cache access latency and cache-line size (64 bytes)
- NUMA topology — cross-node memory access penalty
- False sharing in concurrent structures and how to detect it
- Memory allocator behavior under load (glibc malloc, jemalloc)
- Lock contention detection — `perf`, `strace`, `futex` waits
- Async I/O vs thread pools — when each wins
- Event loop starvation and how to find it

</details>

<details>
<summary><b>🔒 Auth & security engineering</b></summary>
<br/>

- JWT revocation without a blacklist: Redis cutoff-based authentication
- DB-tracked refresh token JTIs and rotation strategies
- RBAC with role-ancestry caching — avoiding permission-check waterfalls
- Permission factory patterns in DRF
- Session security design — what "stateless" actually costs you

</details>
